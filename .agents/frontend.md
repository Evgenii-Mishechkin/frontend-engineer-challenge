# AI Usage — Frontend

## Инструмент

**Cursor IDE + Claude Sonnet** (встроенный агент Cursor).  
Использовался как интерактивный пар-программист в режиме диалога на протяжении всей разработки frontend-части.

---

## Что делал AI

### Генерация кода

- Первичный скаффолдинг компонентов (шаблон + script setup + scoped styles) по описанию задачи
- Реализация composables (`useAsyncFormSubmit`, `useForgotPasswordRequest`, `useLoginForm`, `useRegisterForm`)
- Написание CSS с дизайн-токенами по ссылкам на Figma-макеты
- Настройка тестового окружения (Vitest, vite.config.ts, tsconfig, setup.ts, withSetup.ts)
- Написание unit и component тестов (51 тест в 6 файлах)
- Рефакторинг: объединение `AuthPrimaryButton` + `AuthNeutralButton` в один компонент с `variant`

### Анализ

- Чтение Go-кода бэкенда (`resolver.go`) для понимания anti-enumeration паттерна
- Объяснение почему `success: true` приходит всегда и как читать `token` как сигнал
- Аудит кодовой базы с рекомендациями по упрощению и удалению мёртвого кода

### Итеративные правки

- Доработка стилей по скриншотам (цвета, отступы, max-width)
- Исправление поведения `RouterLink` + scoped CSS через `custom` слот
- Устранение TypeScript-ошибок

---

## Что делал разработчик самостоятельно

- Выбор архитектурного подхода (Feature-Sliced Design)
- Принятие всех ключевых решений: как обрабатывать ошибки полей, где граница между widget и feature, какие компоненты делать shared
- Code review каждого предложения AI перед применением
- Финальные решения по UX: показывать ли inline-ошибку при несуществующем email (vs. anti-enumeration), структура mobile gate
- Верстка части компонентов без участия AI
- Все решения о структуре файлов и именовании
- Диагностика проблем, которые AI не мог воспроизвести (runtime-поведение в браузере)

---

## Ограничения и валидация

- Каждый сгенерированный фрагмент кода проверялся в браузере вручную
- AI несколько раз предлагал решения, которые не работали с первого раза (RouterLink + scoped styles, peer-зависимости Yarn 1 + Vite 8) — разработчик диагностировал и корректировал подход
- Тесты запускались и проверялись локально: `yarn test`
- Финальная сборка проверялась через `yarn build`

---

## Вывод

AI использовался как инструмент ускорения реализации. Архитектурные решения, code review, верификация результата и контроль качества оставались за разработчиком.
